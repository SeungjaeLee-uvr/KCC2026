15 Minute Presentation

협업 XR 저작을 위한
객체 단계 버전 관리 워크플로우

Object-Level Version-Control Workflows for Object Editing in Collaborative XR Authoring

Unity 6000.3.10f Meta Quest 3 Standalone XR Prototype On-device JSON Repository

이승재, 백민주, 우운택 · KAIST · UVR Lab

Motivation

XR 저작은 쉬워졌지만, 변경 이력 관리는 아직 약합니다

01

직접 조작 중심

사용자는 XR 공간에서 물체를 이동, 회전, 크기 변경, 색상 변경하며 빠르게 아이디어를 실험합니다.

02

반복 수정 증가

디자인, 교육, 트레이닝, 콘텐츠 제작에서는 같은 객체를 여러 사람이 여러 번 다듬습니다.

03

덮어쓰기 위험

여러 사용자가 독립적으로 수정하면 어떤 상태가 최신인지, 어떤 변경을 합칠지 판단하기 어렵습니다.

Problem Gap

기존 방식은 동시 편집에는 강하지만,
비선형 협업에는 부족합니다

Linear undo / redo

짧은 개인 작업 흐름에는 유용하지만, 여러 대안 경로를 보존하거나 나중에 병합하기 어렵습니다.

Real-time synchronization

동시 작업의 상태 공유에는 적합하지만, 비동기적으로 만든 변경안을 검토하는 절차는 약합니다.

Needed workflow

객체 변경 이력을 추적하고, 대안 브랜치를 보존하며, 충돌을 비교하고 선택적으로 병합하는 구조가 필요합니다.

Idea

소프트웨어 Git의 개념을
XR 객체 편집 워크플로우로 옮깁니다

Git Concepts
Immersive Object Editing
Commit객체 상태 스냅샷
Branch대안 수정 경로
Pull Request변경 검토 요청
Merge Conflict속성 단위 충돌 비교

System Overview

제안 시스템은 객체 단위 저장소와
XR 내부 제어 UI로 구성됩니다

Shared Virtual Object

position, rotation, scale, color, interactable, sensitivity를 객체 상태로 캡처합니다.

In-world Control UI

commit, branch, PR, merge, checkout을 XR 공간에서 실행합니다.

JSON Repository

commit, branch, PR, conflict 정보를 기기 내부에 저장합니다.

Repository Model

전체 장면 대신, 의미 있는 객체 상태만 버전 관리합니다

RepositoryData

repositoryName, activeBranchName, commits, branches, pullRequests를 보존합니다.

CommitNode

parentCommitId, branchName, object state, changedAttributes를 저장합니다.

ObjectState

위치, 회전, 크기, 색상 같은 저수준 속성을 캡처하고 다시 적용합니다.

핵심 효과: 전체 scene snapshot보다 가볍고, 사용자가 말할 수 있는 수정 단위와 더 가깝습니다.

Commit & Branch

기존 진행을 깨지 않고,
과거 상태에서 새 대안을 만들 수 있습니다

Branch lanes

main branch와 sub branch를 공간적으로 분리해 lineage를 훑어볼 수 있습니다.

Detached commit

과거 commit에서 수정하면 새 branch를 자동 생성해 기존 진행을 덮어쓰지 않습니다.

Pull Request & Merge

수정안은 바로 반영되지 않고,
검토 가능한 PR로 통합됩니다

1Branch Head

대안 객체 상태가 준비됩니다.

2Pull Request

main head와 branch head의 변경 속성을 요약합니다.

3Three-way Diff

base, main, branch를 비교해 충돌 여부를 판단합니다.

4Merge Commit

충돌이 없거나 해결되면 main에 새 commit을 생성합니다.

Conflict Resolution

충돌은 텍스트 목록만이 아니라,
XR 공간의 미니어처 비교로 보여줍니다

Main vs PR Head

두 revision을 side-by-side miniature로 비교해 변경 결과를 공간적으로 확인합니다.

Attribute choices

Rotation, Scale 등 충돌 속성별로 Main 또는 Branch 값을 선택합니다.

Prototype Implementation

프로토타입은 Meta Quest 3에서
standalone XR 앱으로 구현했습니다

VersionControlManager

repository 초기화, commit, branch, PR, merge를 조정하는 핵심 컨트롤러입니다.

StateDiffUtility

객체 속성 비교와 base-main-branch 기반 conflict detection을 수행합니다.

JsonRepositoryStorage

repository_autosave.json을 Application.persistentDataPath에 저장합니다.

ConflictPreviewController

main head와 PR head를 side-by-side miniature로 시각화합니다.

Demo Plan

발표 데모는 하나의 객체가 어떻게
이력을 갖는지 보여주면 됩니다

  1. 초기 상태: repository가 main branch와 initial commit을 만듭니다.
  2. 수정과 commit: 객체를 이동하거나 색을 바꾸고 현재 상태를 commit합니다.
  3. branch 실험: branch를 만들어 다른 수정안을 만듭니다.
  4. PR 생성: main과 branch의 변경 속성을 비교합니다.
  5. conflict review: 미니어처와 텍스트 요약을 보며 값을 선택합니다.
  6. merge: 해결된 상태가 main의 새 merge commit으로 저장됩니다.

Contribution

Limitations & Future Work

현재는 feasibility prototype이며,
다음 단계는 확장성입니다

Current limitation

현재 시스템은 단일 공유 객체에 집중합니다. 실제 XR 저작에서는 여러 객체와 관계가 함께 바뀝니다.

Attribute level

현재는 position, rotation, scale, color 같은 저수준 속성이 중심입니다.

Next work

multi-object history, semantic edit representation, multi-user evaluation으로 확장할 수 있습니다.

Conclusion

XR 협업에서도 변경 이력은 공간 안에서 이해되고
조작될 수 있습니다

제안 시스템은 object-level versioning, branch-based exploration,
pull request review, conflict-aware merge resolution을
XR 저작 환경 안으로 가져옵니다.
